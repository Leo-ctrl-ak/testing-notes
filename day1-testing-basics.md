# Day 1：软件测试基础概念

## 1. 什么是软件测试？
软件测试是通过手动或自动化的方式，验证软件功能是否符合需求、是否存在缺陷的过程。

## 2. 软件测试的核心目的
- 尽早发现并修复Bug，降低上线后的风险
- 验证软件功能是否符合需求规格
- 确保软件的稳定性、可用性和安全性

## 3. 测试与调试的区别
- 测试：发现缺陷（由测试人员执行）
- 调试：修复缺陷（由开发人员执行）

## 4. 结合我的安全背景
在功能测试的基础上，我会额外关注软件的安全性缺陷，比如：
- 输入校验是否完善
- 是否存在SQL注入、XSS等安全漏洞
- 数据传输是否加密
